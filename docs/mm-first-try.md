# モジュラモノリス 最初の一歩 メモ

issue: [#123](https://github.com/yussak/modular-monolith-practice/issues/123)

「とりあえず作る」フェーズの判断メモ。重い決定は ADR に切り出し、それ以外はここで決めて済ませる。

## 方針

- 重い決定（後で覆すと痛い・思想に関わるもの）は ADR にする
- 規約・運用ルール系はこのファイルで決める
- 決まったら都度追記、覆すときは打ち消し線ではなく書き換えてよい（ADRと違って記録性は重視しない）

## ADR にするもの（未決）

判断が必要になった時点で ADR を切る。

- [ ] モジュールの切り方（境界の定義）
- [ ] モジュールの実装方式（名前空間 / Packwerk packs / Rails Engine）
- [ ] Packwerk の `enforce_dependencies` / `enforce_privacy` をいつ true にするか
- [ ] モジュール間通信方式（同期メソッド呼び出し / イベント駆動 / 併用）
- [ ] DB 分離方針（単一 DB / schema 分離 / JOIN 可否 / 外部キー可否）
- [ ] 既存コードの移行戦略（一括 / Strangler Fig）
