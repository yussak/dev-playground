# 在庫管理機能 要件メモ

GitHub issue #99 の要件をまとめる作業ファイル。

## 要件（issue #99 のコメントで yussak が「要件（確定）」と記載したもの）

- 在庫の設定・管理は出品者が在庫管理専用画面で行う（商品登録・編集とは分離）
- 商品登録時の在庫は初期値 0
- 在庫切れの商品は注文不可・カート追加不可、一覧には「在庫切れ」表示を継続する
- 注文時に在庫チェックを行い、在庫を減算する
- キャンセル時に在庫を戻す
- カート追加時に在庫チェックを行う
- 注文確定時に在庫の再チェックを行う
- カートに入れている間に在庫が 0 になったユーザーへの通知（方式は TBD）

## 要件（会話で合意したもの）

- 在庫数は商品オプション単位で持つ
- 在庫切れ表示も商品オプション単位で行う
- 商品全体を「在庫切れ」として扱うのは、その商品の全商品オプションが在庫 0 のときのみ
- 在庫を編集できるのは出品者本人のみ
- 在庫数の取りうる範囲は 0 以上 999,999 以下
- 購入者画面では、在庫数が 5 以下のとき「残り N 点」と表示する
- 注文時に在庫を減らすタイミングは注文確定時とする
- カート追加時に在庫の仮押さえは行わない
- 在庫管理画面では、在庫数の絶対値上書きと増減の両方を提供する
- 在庫切れ表示は商品一覧・商品詳細・カートの 3 画面で行う
- フロント側は画面レンダリング時に在庫情報を表示する
- 在庫切れの商品オプションについては、カート追加ボタンを非活性にする
- API は在庫切れの商品オプションへのカート追加リクエストをエラーで返す（フロント側の制御をすり抜けたケースのガード）
- 注文ボタン押下時に在庫を再チェックし、結果に応じて次のように振る舞う
  - 全アイテム在庫あり: 注文成立、「購入完了しました」を表示
  - 一部アイテムが在庫切れ: 在庫がある分のみ注文成立、「購入完了しました」と在庫切れアイテムのリスト（「以下の商品は在庫切れのため購入できませんでした」）を併記して表示
  - 全アイテム在庫切れ: 注文不成立、在庫切れアイテムのリストのみ表示
- 注文確定時に在庫切れだったアイテムは、カート内の「現在購入できません」セクションへ移動する
- カート画面のレンダリング時にも在庫を確認し、在庫切れになっているアイテムを「現在購入できません」セクションへ自動移動する
- 注文確定時のこのシナリオで非同期通知は行わない（画面表示で完結）
- 注文確定で実際に購入が成立したアイテムはカートから削除する
- 「現在購入できません」セクションに移動したアイテム（在庫切れで購入できなかった分）はカートに残す
- ユーザーは出品者の承認なしに注文をキャンセルできる
- キャンセル時は在庫数を自動加算する
- 出品者の在庫編集は「画面を開いたときの値からの差分」として扱い、保存時に DB の現在値にその差分を適用する（注文確定やキャンセルで在庫が動いていても、差分が合算されるため再操作は不要）

## やらないこと（issue #99 に記載）

- 在庫変動履歴
- 在庫切れアラート（閾値設定）
- 在庫が戻ったときの通知

## 将来の拡張で再検討する項目

- 発送機能が追加されたときに、注文時の在庫減算タイミングを「注文確定で引当 → 発送で実減算」の二段階に再設計する
- タイムセールや数量限定販売などの希少性が高い商品体験を導入する場合は、カート追加時の仮押さえを検討する
- admin による出品停止機能（在庫機能とは別建てで管理する想定）
- 通知連携（在庫 0 時のカート保持ユーザーへの通知など）は、通知機能本体が用意できたタイミングで在庫機能との連携を別タスクとして設計する

## テストで担保したいケース

- 注文確定時に在庫が足りなかったケース（複数ユーザーがすり抜けで競合した場合の挙動）
